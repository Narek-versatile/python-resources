Basic Search Engine

Project Overview
Develop a Basic Google Search Engine that can index a collection of web pages and allow users to perform keyword-based searches to retrieve relevant results. This project aims to introduce students to fundamental concepts of search engines, including web page indexing, keyword extraction, search algorithms, and result ranking.
Objectives
By completing this project, students will:
Understand Search Engine Fundamentals: Learn how search engines index and retrieve information.
Implement Text Processing: Utilize Python for text parsing and processing.
Build Data Structures for Indexing: Create efficient data structures to store and manage indexed data.
Develop Search Algorithms: Implement algorithms to search and rank web pages based on user queries.
Handle File Operations: Manage reading from and writing to files for indexing web pages.
Apply Object-Oriented Programming (OOP): Design classes and objects to structure the search engine components.
Enhance Problem-Solving Skills: Tackle real-world problems related to information retrieval and data management.
Project Components
Indexer: Processes web pages to extract and store keywords.
Search Engine: Accepts user queries and retrieves relevant web pages.
Ranker: Sorts search results based on relevance.
Project Structure
The project will be organized into the following Python modules/files:
search_engine.py: Contains the main classes and methods for indexing and searching.
web_pages/: A directory containing text files that simulate web pages.
main.py: Provides a user interface to interact with the search engine.
requirements.txt: Lists any external Python packages needed (optional).

3. Implementing the Search Engine (search_engine.py)
This module contains the BasicSearchEngine class responsible for indexing web pages and handling search queries.
Explanation of search_engine.py:
__init__ Method: Initializes two dictionaries:
index: Maps each word to a list of web page filenames containing that word.
web_pages: Stores the content of each web page.
index_web_pages Method:
Iterates through all .txt files in the specified folder.
Reads and processes each file, converting content to lowercase and extracting words using regular expressions.
Updates the index and web_pages dictionaries accordingly.
Handles potential errors like missing folders.
search Method:
Takes a user query, splits it into individual words, and searches the index for these words.
Maintains a results dictionary to track the number of query words found in each web page.
Sorts the results based on relevance scores.
display_search_results Method:
Formats and displays the sorted search results to the user.
Indicates when no results are found.
